Unidade 1 — Conceitos e Arquiteturas

Teoria Rápida e Conceitos Fundamentais

Um Sistema Distribuído é uma coleção de computadores independentes interligados via rede, que para o usuário, aparenta funcionar como um único sistema. Eles podem ter hardwares e sistemas operacionais diferentes, mas trabalham juntos.

O elemento central é o Middleware, a camada que fica entre o software e o sistema operacional, responsável por gerenciar comunicação, dados e abstrações.

Os principais objetivos incluem: ocultar a distribuição dos recursos, oferecer alta disponibilidade, e permitir acesso simples e transparente.

Arquiteturas Computacionais

  • Cliente-Servidor: servidor oferece serviços; clientes consomem.
  • Ponto a Ponto (P2P): todos os nós têm o mesmo papel.
  • Arquitetura Descentralizada: híbrido de Cliente-Servidor e P2P, usada em blockchains.

Casos de Uso

  • Ocultação da Complexidade: vários servidores agindo como um.
  • Confiabilidade: servidores assumem carga se um falhar.
  • Sistemas Fracamente Acoplados: comunicação via rede, alta flexibilidade.
  • Middleware: integra camadas de Aplicação, Middleware e SO.

Unidade 2 — Objetivos, Desafios e Modelos

Teoria Rápida

Os três objetivos principais são:

  • Compartilhamento de Recursos
  • Confiabilidade (Teorema CAP)
  • Desempenho

Desafios

  • Abertura
  • Concorrência
  • Escalabilidade
  • Resiliência / Tolerância a Falhas
  • Heterogeneidade
  • Transparência

Casos de Uso

  • Escalabilidade Automática utilizando Chef/Ansible.
  • Granularidade permitindo execução paralela eficiente.

Unidade 3 — Virtualização e Conteinerização

Virtualização

A Virtualização cria versões virtuais de hardware, redes e armazenamento. Máquinas Virtuais permitem rodar múltiplos sistemas operacionais em uma única máquina física.

Conteinerização

Contêineres são leves, isolados e compartilham o kernel do SO. Docker e LXD são as principais engines. Contêineres são base de microsserviços e orquestração com Kubernetes e Docker Swarm.

Casos de Uso

  • Criação de VMs no VirtualBox.
  • Criação de cluster com Docker Swarm.
  • Serviço Apache replicado via docker service create --replicas=5.

Unidade 4 — Segurança e Comunicação

Segurança

  • Confidencialidade
  • Integridade
  • Autenticidade
  • Não Repúdio

Ameaças

  • Leakage (Vazamento)
  • Tampering (Falsificação)
  • Vandalism (Vandalismo)

Comunicação (Sockets)

  • TCP — Confiável
  • UDP — Rápido
  • Primitivas: Socket, Bind, Listen, Accept, Connect...

RPC / RMI

  • Uso de interface Remote em Java.
  • Servidor usa rebind.
  • Cliente usa lookup.
